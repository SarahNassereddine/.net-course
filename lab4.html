Lab 4

Arithmetic Operations Web Application

Build a Razor Pages web application that processes arithmetic operations via query strings and route parameters, with a service layer, EF Core persistence, and a Minimal API history endpoint.

1

Objective

Create a web application that processes various arithmetic operations based on parameters received either through query strings or route parameters. This application will be developed using ASP.NET Core with Razor Pages. Postman may optionally be used to send requests and test the application.

🎯
What You'll Practice
Razor Pages with named page handlers (OnGet…)
Route parameters and query string binding
Route constraints for input validation
Service layer with Dependency Injection
EF Core In-Memory provider and history logging
Exposing a Minimal API endpoint alongside Razor Pages
2

Functional Requirements

The application must support the following six arithmetic operations, grouped into three Razor Pages:

➕ SumMul Page
SUMSum of three integers
MULProduct of three integers
📊 MinMax Page
MINMinimum of three integers ≥ 0
MAXMaximum of three integers ≥ 0
🔢 LcmGcd Page
LCMLeast Common Multiple (2 ints > 0)
GCDGreatest Common Divisor (2 ints > 0)
Route Constraints: Validate input directly in route templates. Sum/Mul accept int; Min/Max accept int values ≥ 0; GCD/LCM accept int values > 0. Test with non-matching routes to ensure graceful handling.
Default Values: For Sum and Mul/Min/Max, if fewer than the required parameters are passed, page handlers must assign sensible default values.
Layout Customization: Modify _Layout.cshtml to include a customized header and footer. Remove unrelated links and menus.
3

Razor Pages & Handlers

Implement three Razor Pages, each with specific handler bindings as described below.

1

SumMul Razor Page

Handles SUM and MUL operations. Accepts three route parameters via GET request and displays the summation and multiplication of the received values.

Handlers
OnGetSum— computes the sum of three values
OnGetMul— computes the product of three values
Binding
Handler name passed via route parameter. Binding done via page handler parameters.
2

MinMax Razor Page

Handles MIN and MAX operations. Accepts three route parameters via GET request and displays the minimum and maximum of the received values.

Handlers
OnGetMin— returns the minimum value
OnGetMax— returns the maximum value
Binding
Handler name passed via query string parameter.
3

LcmGcd Razor Page

Handles LCM and GCD operations. Accepts two route parameters via GET request and displays the Least Common Multiple and Greatest Common Divisor.

Handlers
OnGetLCM— computes the Least Common Multiple
OnGetGCD— computes the Greatest Common Divisor
Binding
Handler name passed via route parameter or query string. Binding done via page handler properties.
4

Database Requirements

💾 Persist Every Call
Create an ArithmeticRequest entity to log each operation, its operands, result, and timestamp.
🗄️ EF Core In-Memory Provider
Install Microsoft.EntityFrameworkCore.InMemory (v 8.0.16) and wire up an AppDbContext that includes a DbSet<ArithmeticRequest>.
🔧 History Service
Implement an interface IArithmeticHistoryService whose concrete class receives the binding model, converts it to an entity, and saves it via the DbContext.
🌱 Seed Demo Rows at Startup
Inside Program.cs, inject a scoped AppDbContext; when the table is empty, insert a few sample records to simplify testing.
🌐 Minimal API History Endpoint
Create a route — e.g. GET /api/requests — that returns the full contents of the ArithmeticRequest table so you can verify persistence with Postman or a browser.
5

Entity Model

Use the following ArithmeticRequest class as your persistence model:

C#ArithmeticRequest.cs
using System;
using System.ComponentModel.DataAnnotations;

public class ArithmeticRequest
{
    [Key] // primary key (autoincrement)
    public int Id { get; set; }

    [Required] [MaxLength(12)] // e.g. "SUM", "GCD", "LCM"
    public string Operation { get; set; } = default!;

    // Operands — nullable to handle 2- or 3-argument ops
    public int? A { get; set; }
    public int? B { get; set; }
    public int? C { get; set; }

    [Required] // store numeric or composite result as text
    public string Result { get; set; } = default!;

    // Automatically stamped when the row is created
    public DateTime AtUtc { get; set; } = DateTime.UtcNow;
}

Technical Notes

binding
Route Data vs. RouteData.Values
To extract route parameters in a Razor Page, you could use RouteData.Values["parameterName"], but this lab requires using proper binding concepts (handler parameters or handler properties).
service
Service Classes & DI
All arithmetic logic must be encapsulated within service classes injected into Razor Pages using Dependency Injection.
test
Testing Non-Matching Routes
Attempt to send requests where no route template matches, to ensure the application handles unmatched routes gracefully.
