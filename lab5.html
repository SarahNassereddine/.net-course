Lab 5

CV Application — Razor Pages

Build a complete ASP.NET Core Razor Pages application for managing Curriculum Vitae data — with a multi-field form, server-side validation, EF Core persistence, photo upload, and a summary page.

Overview

Develop a simple ASP.NET Core web application for managing Curriculum Vitae (CV) information using Razor Pages. The application allows users to enter personal information through a form, validate the entered data, save it in a database, and display a summary page.

🎯
Key Technologies
Razor Pages with Tag Helpers (asp-for, asp-page, asp-validation-for, asp-items)
Dependency Injection with a clean service layer
Server-side validation with Data Annotations and ModelState
EF Core Migrations for database persistence
Separate Binding Model and View Model
File upload for profile photo

Create a home page that welcomes users to the CV application.

🏠 Index Page Requirements
Display a welcome message: "Welcome to our CV Application"

Part 3 — Validation Requirements

Implement proper server-side validation using Data Annotations and Razor Pages validation features.

📧 Email Validation
The email field must contain a valid email address. The Email Confirmation field must exactly match the email field.
🔒 Password Validation
The password must contain letters, digits, and symbols, and be at least 8 characters long.
🔢 Verification Operation (Captcha)
Display two random numbers with a mathematical operation (addition), for example: 5 + 3 = ?. The user must enter the correct result. If the answer is incorrect, an error message must be displayed.
⚠️ Required Fields & Photo
Important fields must not be left empty. The user must also be able to upload a personal photo via the file input.

Part 4 — Summary Page

After a successful form submission, the application must validate all data, save it to the database, and redirect to a Summary Page.

Summary Page Must Display
✓ All entered CV information
✓ The uploaded profile photo
✓ A Random Number value
Random Number
The random number must be received through a route parameter and displayed on the Summary Page.

Part 6 — Technical Requirements

📄 1. Razor Pages Only
The entire application must be implemented using ASP.NET Core Razor Pages only.
💉 2. Dependency Injection
Create a service responsible for saving CV data and inject it into Razor Page Models using Dependency Injection.
🧩 3. Separate Models
Use two distinct models: a Binding Model for receiving form data, and a View Model for displaying information.
🏷️ 4. Tag Helpers
All forms and navigation elements must use ASP.NET Core Tag Helpers: Form, Input, Validation, Anchor, and Select Tag Helpers.
✅ 5. Server-Side Validation
All validations must be implemented on the server side using Data Annotations and ModelState validation.
🗂️ 6. Clean Project Structure
Organize the project using the following folder structure:
Pages/ Models/ Services/ Data/ wwwroot/uploads/
