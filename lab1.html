Lab 1

Student Management API

Build a complete RESTful API for managing student records with validation, filtering, and error handling.

1

Lab Overview

In this lab, you'll create a Web Application that implements a complete student management system. You'll work with records, dictionaries, endpoints, and filters to build a robust API.

🎯
Learning Objectives
Build RESTful endpoints with proper HTTP methods and status codes
Implement custom validation filters using separated classes
Handle errors gracefully with Problem Details responses
Test your API using Swagger/OpenAPI documentation
2

Data Model Requirements

Create a record named StudentList with the following structure:

C#Student.cs
public class Student { public string Name { get; set; } public int Age { get; set; } public string Nationality { get; set; } public decimal Grade { get; set; } } public record StudentList { public Dictionary<int, Student> Students { get; set; } = new(); }
⚠️
Important Note
Don't forget to add {get; set;} for each property to ensure accessibility throughout your application.
3

API Endpoints

Implement the following endpoints for your Student Management API:

GET Endpoints

EndpointGET /api/students
Purpose: Display all students with their properties
Returns: List of all students in the dictionary
Status: 200 OK
EndpointGET /api/students/{id}
Purpose: Get student info by ID (route parameter)
Returns: Student object or Problem Details if not found
Status: 200 OK or 404 Not Found
EndpointGET /api/students/average
Purpose: Calculate the average grade of all students
Returns: Decimal value representing the average
Status: 200 OK
EndpointGET /api/students/summary
Purpose: Return Id, Name, and Nationality of all students
Returns: Simplified list with selected properties only
Status: 200 OK

POST/PUT Endpoints

EndpointPOST /api/students/{id}
Purpose: Add a new student to the dictionary
Body: Student object (JSON)
Returns: URL to the created student or Problem Details if ID exists
Status: 201 Created or 409 Conflict
EndpointPUT /api/students/{id}
Purpose: Modify an existing student
Body: Updated student object (JSON)
Returns: Success message or Problem Details if not found
Status: 200 OK or 404 Not Found
4

Validation Filters

Implement two custom filters to validate incoming data:

ID Validation Filter

✨
Filter Requirements
Create validation logic in a separated class
Apply to the POST endpoint for adding students
Validate that ID is positive and less than 1000
Return Problem Details with customized message on failure
C#IdValidationFilter.cs
public class IdValidationFilter { public static bool IsValidId(int id) { return id > 0 && id < 1000; } }

Grade Validation Filter

✨
IEndpointFilter Implementation
Use IEndpointFilter interface
Apply to endpoints that accept student data
Validate that Grade is between 0 and 100
Return appropriate Problem Details on validation failure
C#GradeValidationFilter.cs
public class GradeValidationFilter : IEndpointFilter { public async ValueTask<object?> InvokeAsync( EndpointFilterInvocationContext context, EndpointFilterDelegate next) { var student = context.GetArgument<Student>(0); if (student.Grade < 0 || student.Grade > 100) { return Results.Problem( "Grade must be between 0 and 100", statusCode: 400 ); } return await next(context); } }
5

Testing with Swagger

Once you've implemented all endpoints and filters, test your application thoroughly using the built-in Swagger UI.

🧪
Testing Checklist
✓ Test retrieving all students
✓ Test retrieving a specific student by valid/invalid ID
✓ Test adding a student with valid/invalid ID
✓ Test adding a student with valid/invalid grade
✓ Test updating existing and non-existing students
✓ Test calculating the average grade
✓ Verify all error messages display correctly
💡
Pro Tips
Use Swagger's "Try it out" feature to test each endpoint interactively
Pay attention to response status codes - they should match the specification
Test edge cases like boundary values (ID = 0, 999, 1000; Grade = -1, 0, 100, 101)
Ensure Problem Details responses include helpful, user-friendly messages
