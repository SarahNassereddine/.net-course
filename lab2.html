This comprehensive lab explores various model binding scenarios in ASP.NET Core. You'll work with simple types, complex objects, collections, and implement multiple validation strategies.

Complex Types & Validation

Task 4-5: Person Management

EndpointPOST /api/person-a
Accepts Person data, logs to console, and saves to collection
EndpointPOST /api/person-b
Same as person-a but with validation using IEndpointFilter:
• Id must be between 1 and 1000

Accepts array of Person objects with same validation rules
EndpointGET /api/person/{id}
Retrieve person by ID with validation (Id between 1 and 1000)

Task 7-10: User Management

EndpointGET /api/user-a
Returns dummy User data with hardcoded values
EndpointGET /api/user-b?age=30&username=user1
Filter users by optional age and/or username query parameters
Returns all users if no parameters specified
EndpointPOST /api/user
Add new user with DataAnnotations validation:
• All fields required
• Age between 1 and 100
• Username max 20 characters
Use WithParameterValidation()
EndpointGET /api/users?minAge=25
Fetch users older than specified age
Validate age is between 1 and 100 using AddEndpointFilter
EndpointGET /api/users/{username}
Retrieve user by username (route parameter)
Validate username exists and is max 20 characters
EndpointPOST /api/user/{username}
Update user's email address using route parameter and request body binding

Task 12-15: Product Management

EndpointPOST /api/product
Add product with DataAnnotations validation ensuring all fields are required
EndpointGET /api/products?min=50&max=150
Retrieve products within price range
Validate min < max and both are positive
Return "No products in this range" if empty
EndpointGET /api/products/{title}?description=xyz
Find products containing keyword in title or description
Validate title and description are not empty strings
EndpointPOST /api/product/{title}/{description}?price=99.99

AsParameters Pattern
This technique allows you to bind multiple route and query parameters to a single object, making your handler signatures cleaner and more maintainable.

Task 17: IValidatableObject

🔍
Business Validation
Implement IValidatableObject on the User class for complex business rules:
• Age must be a multiple of 2
• Email must be from domains: gmail.com or ul.edu.lb
C#User.cs (with IValidatableObject)
public class User : IValidatableObject { public string Username { get; set; } public string Email { get; set; } public int Age { get; set; } public IEnumerable<ValidationResult> Validate( ValidationContext validationContext) { if (Age % 2 != 0) { yield return new ValidationResult( "Age must be a multiple of 2"); } var validDomains = new[] { "gmail.com", "ul.edu.lb" }; var domain = Email?.Split('@')[1]; if (!validDomains.Contains(domain)) { yield return new ValidationResult( "Email must be from gmail.com or ul.edu.lb"); } } }
